# 自作言語入力インターフェース 仕様書

## 1. 概要

本プログラムは、独自のスタック型プログラミング言語（以下「StackedLang」）を入力・実行できる、ブラウザ上で動作する JavaScript プレイグラウンドである。

ユーザーがテキストエリアにプログラムを記述して実行すると、言語処理系がそれを解釈し、画面上の「結果」エリアに `<div>`（箱／Box）を生成・操作・アニメーションさせる。Forth 系のスタックマシンを模した後置記法（逆ポーランド記法）で記述する。

- 動作環境: Web ブラウザ（jQuery 3.6.0 / ES Modules を利用）
- ファイル拡張子（サンプル）: `.stackedlang`

## 4. 言語仕様（StackedLang）

### 4.1 基本モデル

- **スタックマシン**: 値はスタック（後入れ先出し）に積まれる。
- **後置記法**: 演算子・命令はオペランドの後に書く（例: `3 4 +` → 7）。
- **トークン区切り**: 1 個以上の空白文字（スペース・改行・タブ）で区切る（`/\s+/` で分割）。改行も区切り文字として扱える。
- **数値リテラル**: 正の整数のみ（正規表現 `/^[0-9]+$/`）。負数・小数のリテラルは存在せず、`-` や `/` 演算で生成する。
- **真偽値**: 比較命令は真で `1`、偽で `0` を積む。`if` は 0 を偽、非 0 を真とみなす。
- **Box ID**: `place` 等で箱を生成すると `0` から始まる連番 ID がスタックに積まれる。ID は内部要素 `id="Box{n}"` に対応。
- **値の種類**: スタックに積まれる値は次の 3 種類。
  - **Number** … 数値。
  - **ID** … `place` / `clone` / `cloneAt` が返す Box の識別子。算術・比較では数値として扱われる（`id 1 +` のような計算が可能）が、種別としては Number と区別される。
  - **Block** … `[ ... ]` でひとまとめにした命令列（`call` / `loop` / `if` で実行する）。

### 4.2 構文解析（parser.js）

`parseProgram(program)` が文字列を受け取り、命令オブジェクトの配列を返す。

- 数値文字列 → `{ type: "number", value: Number }`
- 既知のキーワード → 対応する `type` を持つオブジェクト（下表）
- `[ ... ]` → ブロック。内部を再帰的に解析し `{ type: "block", body: [...] }` として 1 つの値にまとめる
- `-> 変数名` → `{ type: "defineVariable", variableName }`（`->` の直後トークンを変数名とする。無ければ alert）
- 上記いずれにも該当しない語 → `{ type: "variable", variableName }`（変数参照）

#### キーワードと内部 type の対応

| 記述 | type | 記述 | type |
| --- | --- | --- | --- |
| `+` | plus | `clone` | clone |
| `-` | minus | `cloneAt` | cloneAt |
| `*` | mult | `opacity` | opacity |
| `/` | div | `spin` | spin |
| `=` | eq | `shake` | shake |
| `<` | lt | `fade` | fade |
| `>` | gt | `dup` | dup |
| `print` | print | `over` | over |
| `place` | place | `swap` | swap |
| `move` | move | `rot` | rot |
| `rmove` | rmove | `sleep` | sleep |
| `delete` | delete | `call` | call |
| `color` | color | `loop` | loop |
| `resize` | resize | `if` | if |
| `[` `]` | block | `random` | random |
| `->` | defineVariable | | |

### 4.3 命令リファレンス（全 28 種）

表記の `[...]` はスタックの状態（左が底、右がトップ）。`--` の左が消費、右が生成。

| # | 名前 | スタック効果 | 説明 |
| --- | --- | --- | --- |
| 1 | 数値 | `-- n` | 数値をスタックに積む。例: `10 20 30` |
| 2 | `+ - * /` | `a b -- result` | 上位 2 値を取り出して四則演算し結果を積む。例: `3 4 +` → 7 |
| 3 | `= < >` | `a b -- 0\|1` | 上位 2 値を比較し、真なら 1・偽なら 0 を積む。例: `3 5 <` → 1 |
| 4 | `print` | `n --` | トップの値を出力する。画面に大きく表示し、`console.log` にも出力 |
| 5 | `place` | `x y w h -- id` | x,y,w,h で箱を生成し、生成した箱の ID を積む |
| 6 | `move` | `x y id --` | 箱を絶対座標 (x,y) へアニメーション移動 |
| 7 | `rmove` | `dx dy id --` | 箱を現在位置から相対移動 |
| 8 | `resize` | `w h id --` | 箱のサイズを (w,h) へアニメーション変更 |
| 9 | `color` | `r g b id --` | 箱の背景色を rgb(r,g,b) に変更 |
| 10 | `opacity` | `val id --` | 箱の不透明度を変更（0〜1）。小数は割り算で作る。例: `1 2 / id opacity` → 0.5 |
| 11 | `delete` | `id --` | 箱を爆発（パーティクル）エフェクトで削除 |
| 12 | `fade` | `duration id --` | 箱を duration ミリ秒かけてフェードアウト消去 |
| 13 | `clone` | `id -- newId` | 箱を同じ位置に複製し、新 ID を積む |
| 14 | `cloneAt` | `x y id -- newId` | 箱を指定位置 (x,y) に複製し、新 ID を積む |
| 15 | `spin` | `id --` | 箱を 1 回転アニメーション |
| 16 | `shake` | `id --` | 箱を振動アニメーション |
| 17 | `loop` | `n block --` | ブロックを n 回繰り返す。例: `5 [ id spin ] loop` |
| 18 | `if` | `cond block --` | 条件が真（非 0）ならブロックを実行 |
| 19 | `random` | `n -- r` | 0〜n の整数乱数を積む |
| 20 | `sleep` | `ms --` | ms ミリ秒だけ処理を待機 |
| 21 | `->`（変数定義） | `v --` | トップの値を変数へ保存。例: `42 -> x` |
| 22 | 変数参照 | `-- v` | 変数の値をスタックに積む。中身がブロック（配列）でも実行せず値として積む（実行は `call`） |
| 23 | `dup` | `a -- a a` | トップを複製 |
| 24 | `swap` | `a b -- b a` | 上位 2 値を交換 |
| 25 | `over` | `a b -- a b a` | 2 番目の値を複製してトップに積む |
| 26 | `rot` | `a b c -- b c a` | 上位 3 値を回転 |
| 27 | ブロック `[ ]` | `-- block` | 命令列を 1 つの値（配列）としてスタックに積む |
| 28 | `call` | `block --` | スタックトップのブロックを関数として実行する。例: `[ 1 2 + ] call`、`myFunc call` |

> 注: `move`/`rmove`/`resize`/`color`/`cloneAt` などは「id を最後に積む」記述順になっている（命令の直前に id を置く）。スタック実装上、最後に pop されるのがリストの先頭オペランド。

### 4.4 演算のオペランド順序（実装詳細）

スタックから pop する順序に注意が必要（後から積んだものが先に pop される）。

- `minus`: `a=pop, b=pop` → `b - a`（先に積んだ方から引く）
- `div`: 同様に `b / a`
- `eq/lt/gt`: `b (op) a` を評価。`eq` はブロック同士は参照比較、数値・ID は値で比較する
- `place`: pop 順は h, w, y, x。記述順は `x y w h place`
- `color`: pop 順は id, b, g, r。記述順は `r g b id color`

## 5. 実行系仕様（runner.js）

### 5.1 エントリ

```
runInstructions(instructions)  // async
```

- 空のスタック `stack` と実行コンテキスト `ctx` を生成して `runBody` を呼ぶ。
- `ctx = { result: #result の DOM要素, variables: {} }`。実行中に `ctx.currentId`（生成中の箱 ID）が設定される。

### 5.2 Box ID 管理

- モジュールスコープの `currentId`（初期値 0）を `place`/`clone`/`cloneAt` のたびにインクリメント。
- `resetId()` で 0 に戻す（「全てクリア」「全てクリアして実行」で呼ばれる）。

### 5.3 命令の実行（runBody）

- `queue = [...instructions]` を先頭から順に処理。
- **制御構造の展開方式**: `loop` / `if` / `call` は、対象のブロック命令列を `queue.splice(idx, 0, ...body)` で「現在位置の直後へ挿入」して逐次実行する。
  - `loop`: n 回ぶん body を挿入。
  - `if`: condition が真のときのみ body を挿入。
  - `call`: スタックトップのブロック（配列）を pop して挿入。配列でなければ `alert("call expects a block")`。
- **変数参照**: 変数値をそのまま push する（ブロックも実行せず値として積む）。関数として動かすには続けて `call` を使う。
- DOM 操作を伴う命令（move, rmove, resize, fade, sleep）は `await` され、アニメーション完了まで次の命令を待つ。
- `defineVariable`: `ctx.variables[name] = stack.pop()`。
- 不正命令は `alert("unknown instruction")`。

### 5.4 エラーハンドリング

- ほぼ全命令の冒頭でスタック長を検査し、不足時は `alert("stack underflow")` して実行中断（return）。
- `print` はスタックトップを画面に大きく表示し、併せて console.log にも出力する。

## 6. 描画層仕様（interfaces.js）

すべての箱は `id="Box{n}"`（`boxId(id)` で生成）を持つ `<div class="box">`。

| 関数 | 処理概要 |
| --- | --- |
| `place(ctx,x,y,w,h)` | 箱を生成し座標・サイズを設定、`#result` に追加、box-pop アニメ |
| `move(ctx,id,x,y)` | jQuery `animate` で 500ms かけ絶対移動、完了後 box-pop |
| `rmove(ctx,id,dx,dy)` | 現在位置 + (dx,dy) へ 500ms 相対移動、完了後 box-pop |
| `colorBox(ctx,id,r,g,b)` | 背景色を `rgb(r,g,b)` に設定 |
| `resizeBox(ctx,id,w,h)` | 300ms かけてサイズ変更、完了後 box-pop |
| `cloneBox(ctx,id)` | 同一位置・同サイズ・同色・同不透明度で複製 |
| `cloneBoxAt(ctx,id,x,y)` | 指定座標へ複製（内部 `createClone`） |
| `opacityBox(ctx,id,val)` | 不透明度を設定 |
| `spinBox(ctx,id)` | spin-anim（0.6s 1回転）を再適用 |
| `shakeBox(ctx,id)` | shake-anim（0.5s 振動）を再適用 |
| `fadeBox(ctx,id,duration)` | jQuery `fadeOut` 後に DOM から除去 |
| `deleteBox(ctx,id)` | 箱を除去し、中心から 30 個のパーティクルを放射状に飛散させる爆発演出 |

- アニメ再適用は `animation="none"` → `void offsetWidth`（リフロー強制）→ アニメ再設定の定番手法を使用。

  - `shake-anim`: 左右に振動

## 9. サンプルプログラム（examples.stackedlang）

3 つのループ例を収録。

1. **横一列生成**: `0 -> i  20 [ i 38 * 30 + 250 30 30 place -> _  i 1 + -> i ] loop`

3. **回転**: 各箱を spin させ、間に 60ms の sleep を挟む。

## 10. 既知の制約・補足

- 数値リテラルは非負整数のみ。負数・小数は演算で生成する。
- `print` は値を画面に大きく表示し、併せてブラウザのコンソールにも出力する。
- 「例2」ボタンはクリックハンドラ未実装（UI のみ存在）。
- エラー通知は `alert` による（スタックアンダーフロー、未知命令、`->` の変数名欠落など）。
- 変数はグローバル 1 スコープ（`ctx.variables`）。ブロックを変数に入れると関数のように再利用できる。
- 存在しない Box ID を操作した場合、jQuery セレクタが空となり描画系命令は無視される（move 等の `[0]` 参照では例外になり得る）。
